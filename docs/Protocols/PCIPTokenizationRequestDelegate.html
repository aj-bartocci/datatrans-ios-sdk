PCIPTokenizationRequestDelegate

@objc(DTPCIPTokenizationRequestDelegate)
public protocol PCIPTokenizationRequestDelegate

Implement PCIPTokenizationRequestDelegate to be notified when a tokenization request ends. PCIPTokenizationRequestDelegate will notify you about the success, error or cancel state of the processed tokenization request.

  • tokenizationRequestDidFinish(_:tokenizationId:)

    This is called after a tokenization request has been successfully completed. This callback provides details about the tokenization request.

    tokenizationId

    The resulting tokenizationId.
  • tokenizationRequestDidFail(_:error:)

    This is called after a tokenization request fails or encounters an error. Keep in mind that the SDK shows the error to the user before this is invoked. Therefore, this callback can be used to cancel any on-going process involving the tokenization request.

    You may also use the error details provided here and display it the way you want when suppressing the error message within the PCIPTokenizationRequestOptions.
